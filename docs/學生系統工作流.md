# 🎓 學生系統工作流（六庫一流程）

> 2026-07-10 建立；同日完成「名冊為中心」雙向串接與學期成績庫。
> 核心理念：老師只做「記錄」與「匯入」，其餘由系統自動匯流成報告。

## 名冊＝學生系統核心（2026-07-10 串接完成）

名冊已與四大系統建立**雙向 relation**，在 Notion 點開任一學生頁即可看到：
- 「紀錄」→ 班經與學習紀錄庫該生全部紀錄
- 「📚 學期成績」→ 學期成績庫該生期中/期末各科成績、名次與進步獎（唯一成績來源）
- 「🏦 銀行帳」→ 銀行帳本該生全部帳列
- 「📊 學習報告」→ 每週學習報告（新報告由 /class-report 產生時填「學生檔案」relation）

## 資料流總覽

```
👥 學生基本資料（名冊）─────────────────┐
   座號/姓名/查詢碼/職務/週薪/職務說明     │ 個資只填一次，
   ↑ 幹部指派（關於我們頁公開，姓名去識別化）│ 全系統 relation 引用
                                        ▼
📝 班經與學習紀錄庫 ──/class-log── 一句話記錄課堂與生活表現（含金幣影響自動計算）
                                        │
📚 學期成績庫 ──/class-grades── 成績單上傳即匯入（國數社期考/名次/進步獎；平時50%＋期考50%）
                                        │
🏦 班級銀行帳本 ──/class-bank── 週結：發薪（名冊職務週薪）→ 結算獎懲（紀錄庫金幣影響）
                                → 進步獎（學期成績庫進步獎 +20）→ 利息
                                        │
                                        ▼
📊 學生學習報告 ──/class-report── 每週自動彙整：
   五向度分數（紀錄庫）＋考試成績等第與考週級距表（學期成績庫）＋各科建議（紀錄＋成績）
   ＋亮點＋家長協助建議 → 老師檢查 → 勾發布 → 班網加密查詢
```

## 學期成績架構（國、數、社）

- 平時成績 50% ＝ 日常表現 20% ＋ 平時考 10% ＋ 作業 20%；期考成績 50%。
- 期中成績 ＝ 期中平時 ＋ 期中考；期末成績 ＝ 期末平時 ＋ 期末考。
- 學期成績庫一列＝一生一階段（期中/期末），輸入四欄原始分（0–100），
  「○○平時」「○○成績」為 Notion 公式自動計算（勿手填）。
- 定期評量成績單匯入時寫入對應階段列的「○○期考／名次／進步分數／進步獎」
  （第一次定期評量＝期中、第二次＝期末）；「平均」公式自動算三科平均。
- 期中/期末考週的學習報告自動附「考試成績摘要」（本人成績＋班平均＋名次＋全班級距表）。

## 規模與歸檔（帳本每週約 60 筆、學期上千筆）

- skills 查帳一律用「週次標籤＋座號」精準搜尋，不整庫掃描。
- **學期末歸檔**：學期結束後，把上學期帳本帳列與紀錄庫紀錄移到歸檔資料庫
  （或於 Notion 直接以「學期」篩選後搬移／匯出 CSV 備份再刪除），
  保持現役資料庫輕量；學期成績庫與成績庫屬彙總資料、量小，不需歸檔。
