### 日課表怎麼改
在 Notion「**🕐 日課表**」資料庫改，一列就是一節課（星期＋節次＋科目＋教師＋教室）：
- **換科目／改老師名字／改教室**：直接改那一列的欄位。
- **調課**：改「星期」或「節次」即可。
- **停開某節**：取消勾「顯示」（或刪除該列），該格就會變空白。
- 晨掃、早自修、朝會、午休與週三／週五半天等固定時段由網站自動產生，不用建。
- 節次時間、科目底色與下方說明文字寫在 `scripts/sync-notion.mjs` 的 `SCHEDULE_META`，要改再進 repo。
改完等自動同步（07/16/20 時）或到 Actions 手動 Run workflow 立即生效。

### 學生系統怎麼用（2026-07 全面改版：一句話記錄 → 週報告全自動）

**學生基本資料只設定一次**（名冊：姓名/座號/查詢碼/頭貼/職務/週薪），之後平時只做一件事——
在 Claude Code 用一句話記錄，其餘全自動、報告筆筆有據：

| 時機 | 你說一句話 | 系統做的事 |
|---|---|---|
| 平時隨手（10 秒） | 「座號5 數學小考粗心 -1」「記錄：…」 | 寫入紀錄庫（類別/科目/正負向/程度/金幣自動判） |
| 通知類事項 | 「座號16 視力複檢通知已轉交 記一筆」 | 「事務通知」類：不評分不扣幣，只進家長建議 |
| 考完定期評量 | 上傳成績單「匯入成績」 | 寫入 📚 學期成績庫（期考/名次/進步獎；平時＋期考各佔 50% 自動算） |
| 學期中平時成績 | 「登記國語作業成績…」 | 寫入學期成績庫日常/平考/作業欄，公式自動算 |
| 週五放學 | 「class-bank 週結」 | 發薪（照名冊職務）→ 獎懲入帳（照紀錄庫金幣）→ 利息 |
| 週末 | 「產生本週學習報告」 | 全班報告草稿：五向度、等第、亮點、建議**全部引用當週紀錄**，並附「依據紀錄數」 |
| 需要介入時 | 「幫座號N開學習/生活輔導」 | 開輔導個案：困難點（附事證）→可檢核目標→代價設計輔導行為；報告會自動提示候選名單 |
| 課堂當下即時操作 | 開「班級管理工作台」（班網常用網站＞班級事務） | 黑板/加分/分組/考試計時；**不建真實姓名名單**，值得留存的表現回來仍用一句話記錄 |

報告檢查重點：
1. 「學生」欄只填**姓名**（不加座號）；查詢碼與頭貼從名冊自動帶，報告列不用填。
2. 每列有「**依據紀錄數**」——0 或 1 代表該週幾乎沒觀察紀錄，AI 會提醒「連續兩週空白名單」。
3. **期中考、期末考當週**的報告自動加「📝 考試成績摘要」：孩子各科成績、班平均、名次＋全班級距表。
   **考週不是特殊週**：五向度分數與行為觀察照常由當週紀錄生成（考週也照常用一句話記錄），
   成績摘要只是額外附加，不會取代行為向度。
4. 檢查微調 → 勾「**發布**」→ 家長到班網「📊 學習報告」輸入**座號＋查詢碼**查看；多週自動變成週次切換。

開學、期中考、期末考、休業式等重要日程**自動從班網 Google 行事曆讀取**（週次由此推算），
換學期不用改設定；行事曆沒排到時才退回 Notion「⚙️ 網站設定」的學期設定列。

家長端功能：**🖨️ 列印報告**（一張 A4 直式）、**🕶️ 去識別化**（姓名變「王○明」、座號與頭貼隱藏，適合列印留存或展示）。

🔒 隱私設計：報告與頭貼在同步時就用該生查詢碼**加密**，班網上沒有明文；查詢碼每位學生不同，私下發給該家長；沒發布的列完全不會上傳。

### 學生輔導個案怎麼用（🧭 學生輔導紀錄，/class-counsel）

一列＝一個個案，分兩種：**學習輔導**（各科學習困難）與**生活輔導**（偏差態度行為）。
輔導內容屬高敏個資，**永遠不會同步到班網**，只留在 Notion。

**什麼時候開個案**：每週產生學習報告時，AI 會自動偵測並提示「建議開輔導個案名單」
（任一科學期成績 <70／進步分數 ≤−10／兩週內同類負向紀錄 ≥3 筆／程度 3 重大事件）；
你確認後說「**幫座號N開數學學習輔導**」才開案，不會自動開。

**個案內容三要素**（AI 草擬、你確認）：
1. **困難點與行為描述**：必附事證——學習輔導自動掛「來源成績」、生活輔導掛「來源紀錄」relation，不開空案。
2. **具體目標**：可檢核、有期限（如「期末數學學期成績達 70 以上」），拒絕「變得更好」這類空話。
3. **輔導行為**：依「代價設計 v3」三段式——【採取措施】＋【對孩子說的話】＋【備註/升級條件】，
   行為與後果真實因果連結（自然後果＞補償型＞限制型），孩子能自己說出「我做了什麼，所以我必須＿＿」才算設計成功。

**檢核與結案**：說「**輔導檢核**」→ AI 對照新成績/新紀錄檢查目標，
達標→「已達標」（成效評估寫具體數據與有效做法）；部分改善→「持續觀察」；
升級為對抗型→建議轉介輔導室雙軌。在名冊點開該生頁面可看到其所有個案歷程（🧭 輔導 relation）。

### 商店兌換申請怎麼處理（🛒 小小銀行「能看也能買」，2026-07-18 上線）

學生在班網「🏦 小小銀行」打開存摺後，買得起又有庫存的商品會出現「🛒 我要兌換」。
按下只是**送出申請，不會扣款**——扣不扣、什麼時候扣，都由你決定：

1. 打開班網「**🧑‍🏫 教師專區**」（頁尾入口）→「🛒 兌換申請」區塊會列出待處理申請。
2. **✅ 核可**：系統自動幫你——帳本記一筆「消費」扣崑山幣 → 商店庫存 −1 → 申請標「已完成」。
   餘額不足會先警示（可選擇強制核可，餘額變負數）。
3. **❌ 駁回**：填一句原因（例：先完成本週任務），不扣款；孩子下次登入存摺看得到。
4. 處理完按「**⚡ 立即更新班網**」，孩子的存摺與商店庫存才會更新到站上。
5. **購買明細**：同區塊「📜 購買明細」看最近 50 筆；完整搜尋開 Notion「🛒 兌換申請」
   （誰、買什麼、多少幣、何時申請／處理、狀態、備註，永久保留）。

防呆設計：每人同時最多 3 筆待處理申請；價格以 Notion 商店為準（改前端騙不了）；
申請要座號＋查詢碼驗證，別班孩子或路人無法亂送。

⚙️ **啟用前提（一次設定）**：Apps Script 代理需更新到 v2.1——開 script.google.com
→ 原代理專案 → 貼上 repo `scripts/apps-script-proxy-v2.gs` 全部內容 → 部署「新版本」（網址不變）。
並確認 Notion integration 能存取 👥 名冊、🏦 帳本、🏪 商店、🛒 兌換申請 四個資料庫
（最簡單：把「🏫 班級經營中心」整頁分享給 integration）。

### 班級工作台的加分要進報告（🖥️ 工作台回流，2026-07-19 建置｜⏳ 待開學實資料驗證）

課堂上用「班級工作台」（CLASSROOM HUB）按的加分、代幣、出勤、作業狀態，
可以整批倒進紀錄庫，不必再手動補記。做法：

1. 工作台 →「設定」→「**匯出本機資料**」，會下載一個 `.json` 檔。
2. 把檔案丟給 Claude Code，說「**工作台回流**」。
3. AI 列出**待審清單**（只出座號，不出姓名），例如
   「2026-07-18 座號3 發表很有想法 +3〔工作台代幣 +3 → 崑山幣 +15〕」。
4. 你看過、確認沒問題 → AI 才寫進紀錄庫；不確認就不會寫。

**幾個要知道的**
- **不會重複計算**：同一筆加分回流過就不會再出現，可以放心每週匯出一次。
- **代幣就是崑山幣**（2026-07-19 起）：工作台按幾點就是幾點，**1 代幣＝程度 1＝5 崑山幣**。
  你在工作台按 +3，紀錄庫就記程度 3，銀行入帳 +15 崑山幣，尺度和你平常手動記的獎懲金一致。
  清單每筆都會直接標出換算後的崑山幣，覺得某筆該多給或少給，在待審階段直接跟 AI 說。
- **只同步行為獎懲**：薪水、利息、商店消費只存在班級銀行，不會倒回工作台；
  所以**兩邊餘額本來就不一樣**（工作台＝行為點數、銀行＝總資產），不必對帳。
- **只回流「有事的」**：正常出席、作業已完成不會記錄，避免洗版；
  只有遲到／缺席／請假、未繳交／待訂正才進清單。
- **座號會自動雙重核對**：工作台的座號和學生姓名，會拿去跟名冊對照表比對**兩次**。
  兩邊都對得上才直接採用；若座號對得上但姓名對不上（例如名冊換過座號、有轉學生、
  或不小心匯出到別班），該筆會標「**座號待確認**」並寫明原因，請你確認後再入庫；
  兩邊都對不上的學生，整個跳過不入清單，不會亂猜。
  → 平常一切正常時你不會看到任何警告；**一看到警告就代表名冊真的該更新了**。

> ⏳ 目前狀態：所有邏輯已用測試檔驗證通過（含全班 27 人與三種異常情境）。
> 唯一還沒做的是拿你的**真實匯出檔**跑一次，確認欄位名稱與程式預期一致。
> 開學第一次使用時跑一次即可，若有落差 AI 會直接報錯而不會寫錯資料。

## 二、班網什麼時候更新？

- 自動：每天台灣時間 **07:00、16:00、20:00** 同步一次。
- 立即更新（班網首頁按鈕，最方便）：見下方「首頁一鍵更新按鈕」。
- 立即更新（電腦）：到 GitHub repo → **Actions** → 「同步 Notion 資料到班網」→ **Run workflow**。
- 立即更新（手機一鍵）：見下方「手機一鍵更新班網」。

### 首頁一鍵更新按鈕（設定一次，之後在班網首頁直接按）

1. 開 https://script.google.com → 新增專案，貼上 repo 裡 `scripts/apps-script-update-proxy.gs` 的全部內容。
2. 左側「專案設定」→「指令碼屬性」新增兩筆：
   - `GH_TOKEN`：GitHub token（建法同下方步驟 1，可共用同一把）
   - `PASSWORD`：自訂更新口令（按按鈕時要輸入，防止路人亂按）
3. 右上「部署」→「新增部署作業」→「網頁應用程式」→ 執行身分「我」、存取權「任何人」→ 複製 `…/exec` 網址。
4. 到 Notion「⚙️ 網站設定與關於我們」新增一列：項目＝`一鍵更新網址`、內容＝貼上網址。
5. 下次同步後，班網首頁右欄就會出現「⚡ 老師專區」的更新按鈕，輸入口令即可觸發同步。

### 手機一鍵更新班網（iOS 捷徑，設定一次終身受用）

**步驟 1：建一把專用鑰匙（GitHub Token）**
1. 手機或電腦開 https://github.com/settings/personal-access-tokens/new
2. Token name 填 `class-website-sync`；Expiration 選 **No expiration**（或一年，到期再換）。
3. Repository access 選 **Only select repositories** → 勾 `class-website`。
4. Permissions → Repository permissions → **Contents** 設為 **Read and write**，其他不動。
5. 按 Generate token，**複製整串 `github_pat_` 開頭的文字**（只會顯示一次）。

**步驟 2：建 iOS 捷徑**
1. 打開「捷徑」App → 右上 **＋** 新增捷徑，命名「更新班網」。
2. 加入動作：搜尋「**取得 URL 內容**」（Get Contents of URL），設定：
   - URL：`https://api.github.com/repos/flyshan2010/class-website/dispatches`
   - 方法：**POST**
   - 標頭（Headers）加兩條：
     - `Authorization` → `Bearer 貼上剛才的token`
     - `Accept` → `application/vnd.github+json`
   - 請求本文（Request Body）：選 **JSON**，加一個欄位 `event_type` = `sync-now`
3. （可選）再加一個「顯示通知」動作，內容填「班網同步中，約 1 分鐘後生效 ✅」。
4. 完成！可把捷徑加到主畫面或桌面小工具，**一點就同步**。

⚠️ Token 等同這個 repo 的鑰匙，只放在自己手機的捷徑裡，不要貼給別人或貼進聊天群。

## 三、一週三報（每週五 5 分鐘）

1. 在「📰 班級週報」新增一筆，填五欄位：學習重點／班級活動／學生亮點／下週提醒／家長配合事項。
2. 狀態改為「**待產文案**」。
3. 打開 Claude Code 說「**幫我產生本週週報三端文案**」→ AI 會把「班網版／LINE版／FB版」寫回 Notion。
4. 檢查文案 → 複製 LINE 版貼班群、FB 版貼社團 → 勾「LINE已發」「FB已發」。
5. 狀態改「**已發布**」→ 班網週報頁自動出現完整版。

## 四、相簿照片流程

1. Google Drive 建活動資料夾（放在「班級活動照片」母資料夾下）。
2. 資料夾權限設「**知道連結的任何人：檢視者**」。
3. 照片丟進去。
4. Notion「🖼️ 相簿索引」新增一筆：活動名稱＋日期＋資料夾連結＋勾「顯示」。
5. 下次同步後班網就有縮圖牆。

⚠️ 照片上網前請確認學生肖像權同意書；避免單一學生特寫＋全名同框。

## 五、初次設定（只做一次）

### 1. 建 Notion integration（讓 GitHub 能讀資料）
1. 到 https://www.notion.so/my-integrations → 「New integration」。
2. 名稱填 `class-website-sync`，Workspace 選自己的，Capabilities 只需 **Read content**。
3. 複製 **Internal Integration Secret**（`ntn_` 開頭）。
4. 回到 Notion「🏫 班級經營中心」頁面 → 右上「…」→「Connections」→ 加入 `class-website-sync`。

### 2. 建 Google Drive API key（相簿用，可先跳過）
1. https://console.cloud.google.com/ → 建專案 → 啟用「Google Drive API」。
2. 「憑證」→ 建立 API 金鑰 →（建議）限制只能呼叫 Drive API。

### 3. 把金鑰放進 GitHub
GitHub repo → Settings → Secrets and variables → Actions → New repository secret：
- `NOTION_TOKEN` = Notion integration secret
- `DRIVE_API_KEY` = Drive API 金鑰（沒有也能跑，只是相簿沒縮圖）

### 4. 修改班級基本資料
校名、班級、學年度、班級口號、「關於我們」都在 Notion「⚙️ 網站設定與關於我們」維護。
日課表在 Notion「🕐 日課表」資料庫維護。Google 日曆嵌入網址、選單項目與模組顏色在 `data/site-config.json`（技術設定，一般不用動）。

## 六、學期回顧頁

班網的「🎓 學期回顧」頁**全自動**：把已發布的週報（亮點、活動、照片）與相簿彙整成統計數字＋亮點牆＋活動剪影＋每週足跡，不用維護，期末打開就是現成的班級成果展。
